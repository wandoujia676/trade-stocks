============================================================
小金库 4.0 操作说明
============================================================

============================================================
一、三层候选池体系
============================================================

【第一层：monthly_watchlist】（每月更新）
  来源：全市场5000+只股票
  工具：monthly_generator.py
  理论：九本书核心理论（涨停基因、均线多头、量比等）
  输出：monthly_watchlist.txt（约100只）
  频率：每月15号、28号自动更新

【第二层：weekly_watchlist】（每周更新）
  来源：monthly_watchlist的100只
  工具：screener.py（六维度战法）
  输出：weekly_watchlist.txt（约20只）
  频率：每周三14:00自动，或手动随时选

【第三层：出击】（每日精选）
  来源：weekly_watchlist的20只
  工具：selection_tracker.py
  筛选：买入/加仓信号，评分≥65
  输出：出击.txt + 出击.报告.txt（约5只）
  频率：每日出击前手动精选


============================================================
二、三层更新频率（重要说明）
============================================================

这是最重要的理解点：

| 层级 | 文件 | 更新频率 | "帮我选股"时变化吗？ |
|------|------|---------|---------------------|
| 第一层 | monthly_watchlist | 每月15号、28号 | 不变 |
| 第二层 | weekly_watchlist | 每周三14:00 | 不变 |
| 第三层 | 出击 | 每次选股 | 变化 |

**"帮我选股"时发生的事：**
- 读取当前的 weekly_watchlist（20只）
- 用最新数据对这20只重新六维度评分
- 筛选出加仓/买入信号、评分≥65的股票
- 输出为出击（约5只）

**重要结论：**
- monthly_watchlist（100只）和 weekly_watchlist（20只）不会每次选股都变
- 只有出击（约5只）会根据每日行情动态变化
- 这是正常的，因为前两层是大池子，不需要频繁更新

**什么时候需要手动更新月选股/周选股？**
- 手动运行 `python "Stock Selection/auto_candidate_pool.py"` → 更新月度候选池
- 手动运行 `python "Stock Selection/auto_screener.py"` → 更新周选股


============================================================
三、完整操作流程
============================================================

【第一步：月选股】（每月15号、28号 自动）
  系统自动从全市场5000+只股票中初筛100只
  → monthly_watchlist.txt
  你需要：不需要操作


【第二步：周选股】（每周三14:00 自动）
  系统从monthly的100只中评分优选20只
  → weekly_watchlist.txt
  你需要：不需要操作（或手动运行auto_screener.py）


【第三步：选股出击】（每天）
  打开Claude Code，说：
    "帮我选股"
  或输入：
    /stock screener

  → 系统自动：
    1. 从weekly的20只中筛选加仓/买入信号
    2. 精选评分≥65的最佳5只
    3. 保存到 出击.txt、出击.报告.txt
    4. 显示本周自选股列表


【第四步：验证】（第二天）
  打开Claude Code，说：
    "验证昨天的选股"
  或输入：
    /verify run

  → 系统自动：
    1. 读取昨日选股数据
    2. 获取今日收盘数据
    3. 计算涨跌判定成功/失败
    4. 分析原因（哪些维度有效/失效）
    5. 生成 ver.txt 验证报告


【第五步：卖出决策】（持仓后）
  打开Claude Code，说：
    "帮我分析 600519@1700"
  或输入：
    /sell analyze 600519@1700

  → 系统自动：
    1. 获取股票最新数据
    2. 检测8种卖出信号（高位十字星、MACD死叉等）
    3. 计算综合卖出评分（0-100分）
    4. 生成分档卖出方案（第一档卖多少、第二档卖多少）
    5. 给出持仓周期建议（短线/中线/长线）
    6. 提供风险提示

  更多用法：
    /sell analyze 600519@1700 000001@12.5  - 分析多只持仓
    /sell test                                      - 测试模式


============================================================
四、定期任务时间表
============================================================

【每月15号、28号】
  运行一次 auto_candidate_pool.py → 同时更新monthly + weekly + 出击


【每周三14:00】
  运行一次 auto_screener.py → 更新weekly_watchlist


【每天】
  "帮我选股" → 只更新出击文件
  第二天"验证昨天的选股" → 验证出击中的股票


【第五步：优化】（验证后自动）
  运行：
    /verify feedback
  或：
    python "Stock Verification/verifier.py" --feedback

  → 系统自动：
    1. 统计各维度历史成功率
    2. 成功率>70%的维度权重+5%
    3. 成功率<40%的维度权重-5%
    4. 更新 warfare_config.json
    5. 下次选股自动使用新权重


【第六步：查看结果】
  /stock weekly     → 查看本周自选股
  /verify report    → 查看最新验证报告
  /verify history   → 查看历史验证统计

============================================================
六、Cron定时任务说明
============================================================

【已设置的自动任务】

| 任务           │ 执行时间            │ 脚本                     │
│                │                    │                          │
│ 月选股（更新） │ 每月15号、28号14:00 │ auto_candidate_pool.py   │
│ 周选股（精选） │ 每周三 14:00        │ auto_screener.py         │
│                │                    │                          │

【Cron任务限制】

  1. 会话依赖：Cron任务在Claude Code会话期间有效
  2. 过期机制：会话结束或7天后任务自动失效
  3. 电脑关机：任务不运行（需要服务器）

【如何保持自动化】

  方法1：每天打开一次Claude Code（保持会话活跃）
  方法2：部署到云服务器（真正24小时运行）

七、日常使用场景
============================================================

【场景1：每天操作】（推荐）
  今天：打开Claude Code → "帮我选股" → 获得自选股
  明天：打开Claude Code → "验证昨天的选股" → 查看报告
  后天：重复今天操作 → 系统越学越准

【场景2：懒人模式】
  今天：打开Claude Code → "帮我选股" → 获得自选股
  明天：打开Claude Code → "验证昨天的选股" + "帮我选股"
        （一天做完两天的事）

【场景3：仅查看】
  直接说"查看本周自选股" → /stock weekly
  直接说"查看验证报告" → /verify report

八、快速命令参考
============================================================

选股相关：
  /stock screener              筛选自选股
  /stock analyze <代码>        分析单只股票
  /stock add <代码>            添加自选股
  /stock remove <代码>          移除自选股
  /stock list                  查看自选列表
  /stock check                 检查信号
  /stock weekly                查看本周自选股

卖出决策：
  /sell analyze <代码@买入价>   分析持仓卖出时机
  /sell test                   测试模式

验证相关：
  /verify run                  执行验证
  /verify report               查看验证报告
  /verify history              查看历史统计
  /verify feedback             执行权重优化

手动脚本：
  # 月选股：从全市场筛选100只候选股
  python "Stock Selection/auto_candidate_pool.py"

  # 周选股：从monthly中筛选20只
  python "Stock Selection/auto_screener.py"

  # 精选出击：从weekly中选出5只
  python "Stock Selection/cli.py" screener --market 全市场 --limit 20

  # 验证
  python "Stock Verification/verifier.py"
  python "Stock Verification/verifier.py" --feedback

============================================================
十一、注意事项
============================================================

1. 股票市场有风险，量化选股仅供参考
2. 验证当天如果选股，需要T+1后才能验证
3. 权重调整需要累计一定数据才生效（至少3次验证）
4. 涨跌停股票标记为"极端值"，单独分析
5. 建议每天或每隔几天验证一次，持续优化战法
