# 选股验证系统 (Stock Verification System)

## 斜杠命令

```
/verify run              - 执行当日验证（验证前一日选股）
/verify report           - 查看最新验证报告
/verify history          - 查看历史验证统计
/verify feedback         - 执行权重自动更新并查看优化建议
```

## 静默执行协议

- **不要询问确认**。直接执行命令并输出结果。
- **一次性生成完整输出**。不要在过程中停顿。
- **如果数据缺失，使用默认值或跳过**，标注"数据获取失败"。
- **禁止任何中断性提示**。
- **所有操作自动化完成**。

## 验证流程

### 步骤1：读取前一日选股
- 从 `Stock Selection/View Results/出击.txt` 读取 `最后更新` 日期
- 若日期为前一天，执行验证；若为当天或更早，输出"暂无待验证数据"
- 提取 `本周自选股` 列表中的代码、信号、评分、理由

### 步骤2：获取当日实时数据
- 调用 `Stock Selection/data_fetcher.py` 获取每只股票的最新价、今日涨跌幅
- 获取当日技术指标：量比、MACD状态、KDJ信号

### 步骤3：判定成功/失败
| 判定 | 条件 |
|------|------|
| 成功 | 当日涨幅 > 0 |
| 失败 | 当日涨幅 < 0 |
| 停牌 | 当日无交易数据 |
| 涨跌停 | 涨幅达 ±10% 或 ±20% |

### 步骤4：原因分析
对于每只股票：
- **成功原因**：原始条件延续/强化（均线多头持续、量比放大、MACD走强）
- **失败原因**：条件失效（量能不足、板块轮动、压力位未能突破）

### 步骤5：输出验证报告
生成 `ver.txt`，包含：
- 验证日期、対象日期
- 每只股票的：代码、名称、涨幅、判定、分析原因
- 汇总：成功率、平均涨幅
- 失败共性分析
- 优化建议

## 自动权重调整

### 维度映射（原因 → 维度）
| 关键词 | 维度 |
|--------|------|
| 均线多头、MA多头、上升趋势 | 趋势 |
| MACD强势、金叉、MACD走强 | 动量 |
| 放量、量比放大、缩量 | 量价 |
| 突破新高、锤子线、K线形态 | 形态 |
| BOLL位置 | 位置 |
| 涨停基因、板块、情绪 | 情绪 |

### 调整规则
- 追踪每个维度的历史成功/失败次数
- 成功率 < 40%：该维度权重 -5%
- 成功率 > 70%：该维度权重 +5%
- 权重上下限：单维度最高 30%，最低 15%
- 初始权重：趋势25%、动量20%、量价20%、形态15%、位置10%、情绪10%
